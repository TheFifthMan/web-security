# 攻击向量
黑客通过各种手段将恶意代码发送至编译器并且进行执行。

# 安全脆弱点
常见的注入发生于：
1. SQL
2. LADP
3. Xpath
4. NoSQL
5. OS Command
6. XML Parsers
7. SMTP headers
8. expression language
9. ORM

# 可能发生漏洞的地方
1. 用户输入的数据没有进行过滤，验证
2. 动态查询没有使用参数化的方式进行，直接将内容发送至编译器进行执行
3. 恶意代码通过ORM的形式进行构造，从而解析窃取到额外的敏感的数据
4. 代码和恶意数据进行直接拼接，例如sql语句或者系统命令，从而造成执行动态查询，命令执行等动作
